## 1. Demo 简介

**OmniMind Nexus** 是一个基于 A2A（Agent-to-Agent）协议的多智能体协作平台，形态为 Web 网站，纯 2D 像素极简主义美学。

**面向用户：** 需要处理复杂任务的知识工作者——产品经理、创业者、研究者、开发者、学生。你有一个模糊的想法，需要多个领域的"专家"帮你拆解、分析、辩论、最终输出可交付成果。

### 核心功能

**32 个专业 Agent 协同工作** —— 通过 A2A 七层协议（发现→记忆→信用→协商→情感→仲裁→联邦），Router 自动匹配最优 Agent 组合，Planner 制定执行计划并拆解任务，Agent 按任务流水线依次执行，支持多轮交叉讨论和仲裁裁决。

**三大智能引擎（参考 OpenClaw + Hermes 设计）：**

| 引擎 | 功能 | 灵感来源 |
|---|---|---|
| 🧠 MoA 混合智能体 | 多模型并行会诊 + 主任模型综合拍板，回答质量超过单模型 | Hermes MoA |
| ⏰ 定时任务调度器 | cron 定时巡检、自动报告、新闻摘要，驻扎在电脑里的 AI 员工 | OpenClaw 生物钟 |
| 🔧 自动技能创建 | 从对话中自动提取可复用技能，越用越聪明 | Hermes 自我进化 |

**五大智能层（默认全开，无需配置）：**

| 层次 | 功能 |
|---|---|
| 🐝 蜂群智能 | 基于信息素的集体决策机制，群体智能共识 |
| 🧬 基因进化 | 遗传算法优化 Agent 参数，每次任务后自动进化 |
| 📊 知识图谱 | 带遗忘曲线的语义记忆系统，自动提取概念和关系 |
| 🪞 元认知层 | 认知偏差检测 + 自我反思，自动生成反思报告 |
| 👤 用户模型 | 跨会话记忆偏好，自动追踪知识水平、关注领域、输出格式偏好 |

**2D 像素流水线可视化** —— 左侧 Agent 列表 + 中间消息流 + 右侧会话历史，三栏布局。每个 Agent 以独立像素角色出现，实时状态（等待/工作中/完成），流式气泡对话，自动滚动。文件产出以卡片展示，点击即可下载。

**工具调用 + 代码库编辑** —— Agent 可以调用 `codebase_read` 读取项目源码、`codebase_edit` 精确替换修改文件（带路径安全校验和唯一性检查）、`file_write` 创建工作区文件、`generate_document` 生成 docx/xlsx 文档、`web_search` 联网搜索。实现真正的「Agent 间文件交接」。

**编码能力增强** —— 代码语法高亮（关键字紫色、字符串绿色、注释灰色）、一键复制代码块、键盘快捷键（Ctrl+Enter 发送、Ctrl+K 命令面板、Ctrl+L 清空、Ctrl+N 新建、↑↓ 输入历史浏览）、Toast 通知系统。

**多场景页面：**

| 页面 | 功能 |
|---|---|
| 🏠 首页 | 主对话流水线，32 Agent 协作 |
| 📊 仪表盘 | 统计数据、Agent 活跃度 |
| 🎤 辩论赛 | 正反方 Agent 辩论，投票裁决 |
| ☕ 咖啡馆 | 灵感碰撞，创意 brainstorming |
| 🎨 画布 | 视觉工作区，自由拖拽布局 |
| 🗺️ 拓扑图 | Agent 网络拓扑可视化 |
| 📈 知识图谱 | 力导向图展示语义记忆，可拖拽探索 |
| 📚 知识库 | RAG 文档问答 |
| 📁 工作区 | 文件管理，Agent 产出浏览 |
| 💤 梦境 | Agent 潜意识记忆回顾 |
| ⏰ 调度器 | 定时任务管理，cron 自动执行 |
| ⚙️ 设置 | API 配置、模型参数 |

**会话历史管理** —— 自动保存、加载、删除历史会话，session 数据持久化，跨会话用户画像追踪。

**成就系统** —— 使用过程中自动解锁成就，弹窗特效 + 粒子动画，点击关闭。

## 2. Demo 创作思路

**灵感来源：** 我日常工作中经常需要同时面对「市场分析 + 技术方案 + 风险评估 + 文案输出」这种多维度任务。现有的 AI 聊天工具只能单点对话，每次都要手动切换上下文、重新描述需求。我想：能不能让多个 AI 像人类团队一样协作——有人做研究、有人写代码、有人审安全、有人出报告，最后自动汇总交付？

**参考了业界标杆：** 研究了 OpenClaw（龙虾 Agent）的五大核心模块（大脑/五官/手脚/日记本/生物钟）和 Hermes Agent 的 MoA 混合智能体 + 自我进化机制，将这些前沿设计落地到了自己的项目中。

**想解决的问题：** 复杂任务需要多领域知识，但单轮对话无法承载深度协作。用户被迫当"项目经理"手动协调，效率极低。AI 不应该只是问答机器，而应该是一支驻扎在电脑里的 AI 团队。

**设计取舍：** 选择了纯白像素美学（圆点网格背景、2px 直角边框、零阴影、零圆角），因为 A2A 协议本身就像数字世界的"流水线"，像素风格天然契合这种机械感。放弃传统聊天列表，改用三栏流水线布局，因为看一眼就知道「任务进行到哪一步了、谁在等谁、谁在产出什么」。从 2.5D 等距视角改回纯 2D，因为更简洁、更聚焦信息本身。

**开发过程：** 全程使用 TRAE IDE，从最初的像素 UI 统一、到四大功能集成、到 2.5D 风格尝试、再到改回纯 2D、最后加入 MoA/调度器/技能创建/用户模型——每一步都是「说一句话，TRAE 写代码」。最让我惊喜的是 TRAE 会主动处理我没想到的边界情况，比如 stale closure 修复、路径安全校验、cron 表达式解析、client/server 端代码分离等。

## 3. Demo 体验地址

由于 Demo 需要配置 OpenAI API Key（兼容所有 OpenAI 格式的 API，包括 DeepSeek 等），提供两种体验方式：

**方式一（推荐）：拉取项目源码本地运行**

```bash
git clone https://github.com/tbsky166/omnimind-nexus.git
cd omnimind-nexus
echo "OPENAI_API_KEY=sk-xxx" > .env.local
npm install && npm run dev
```

打开 `http://localhost:3000`，在设置页填入 API Key 即可体验。首页默认进入流水线对话模式，导航栏可切换所有功能页面。

**方式二：云服在线部署**

[OmniMind Nexus — 全能型多智能体协作平台](https://你的部署地址)，有 10 元 DeepSeek 额度，陆续充。

**测试版（dev 分支）：** 包含最新功能但可能不稳定。

```bash
git clone -b dev https://github.com/tbsky166/omnimind-nexus.git
```

## 4. 技术栈

- **框架：** Next.js 15 + React 19 + TypeScript
- **动画：** Framer Motion
- **样式：** Tailwind CSS 4 + 自研像素极简主义设计系统
- **AI 协议：** A2A（Agent-to-Agent）七层协议
- **流式通信：** SSE（Server-Sent Events）
- **数据存储：** 文件系统 JSON + localStorage
- **支持模型：** OpenAI / DeepSeek / 所有 OpenAI 兼容 API
